# Phase 3 — Discovery and Backlog Readiness Essentials

## What to Do
- Conduct user research; produce Discovery Findings, Pain Point Map, and User Journey Map
- Shape requirements into acceptance criteria catalog and a ready, prioritized backlog
- Assess data readiness and define AI backlog items with experiment hypotheses
- Provision and verify development environment and CI/CD pipeline
- Run Backlog Readiness Review with PO and Delivery Lead before Gate C

## Who
| Role | Responsibility |
|------|----------------|
| UX / PM | Discovery Findings, Pain Point Map, User Journey Map |
| Product Owner | Acceptance Criteria Catalog, AI Backlog Items sign-off |
| Data Lead | Data Readiness Notes |
| ML Lead / PO | AI Backlog Items |
| PO + Delivery Lead | Readiness Notes (joint sign-off) |

## Evidence Required for Gate C
| Artefact | Template | Evidence Level |
|----------|----------|----------------|
| Discovery Findings | `templates/phase-3/discovery-findings.md.template` | Mandatory (reviewed) |
| Pain Point Map | `templates/phase-3/pain-point-map.md.template` | Mandatory (exists) |
| User Journey Map | `templates/phase-3/user-journey-map.md.template` | Conditional (waivable for internal tooling) |
| Acceptance Criteria Catalog | `templates/phase-3/acceptance-criteria-catalog.md.template` | Mandatory (reviewed) |
| AI Backlog Items | `templates/phase-3/ai-backlog-items.md.template` | Conditional (waivable if no AI in Phase 4) |

| Readiness Notes | `templates/phase-3/readiness-notes.md.template` | Mandatory (approved) |

## Key Skills
- `backlog-management` — use when shaping requirements and prioritizing the backlog in subfase 3.2
- `definition-of-done` — use when agreeing DoD at story and sprint level before delivery starts
- `ai-lifecycle` — use when defining AI backlog items and data readiness in subfase 3.3
- `artefact-authoring` — use when filling Discovery Findings and Acceptance Criteria Catalog templates
- `gate-checklist` — use when preparing the Gate C evidence pack

## Entry Criteria
- Gate B passed; product vision and working model in place

## Exit Criteria
- Backlog Readiness Review passed; Gate C passed; team ready to begin sprint delivery
