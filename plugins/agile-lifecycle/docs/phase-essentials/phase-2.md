# Phase 2 — Inception and Product Framing Essentials

## What to Do
- Establish product vision, goals, and success metrics aligned to the value hypothesis
- Agree governance model, team structure, working agreements, and decision rights
- Produce initial architectural decisions (ADRs) and architecture pack
- Build initial roadmap and release plan aligned to business goals
- Assemble and sign the Inception Closure Pack before Gate B

## Who
| Role | Responsibility |
|------|----------------|
| Product Owner | Product Vision, Product Goal Set |
| PM | Working Model, Governance Model, Role-Responsibility Map, Initial Roadmap, Inception Closure Pack |
| Engineering Lead | Initial Architecture Pack, Initial ADR(s) |
| PM + PO (joint) | Initial Roadmap sign-off |
| Steering Committee | Awareness of Gate B outcome |

## Evidence Required for Gate B
| Artefact | Template | Evidence Level |
|----------|----------|----------------|
| Solution Brief | `templates/phase-2/solution-brief.md.template` | Mandatory (reviewed) |
| Architecture Decision | `templates/phase-2/architecture-decision.md.template` | Mandatory (reviewed) |
| Iteration Plan | `templates/phase-2/iteration-plan.md.template` | Mandatory (reviewed) |
| Gate B Pack | `templates/phase-2/gate-b-pack.md.template` | Mandatory (reviewed) |

## Key Skills
- `phase-contract` — use when creating the Phase 2 contract and validating entry criteria
- `artefact-authoring` — use when creating Product Vision, Architecture Pack, or Roadmap
- `risk-management` — use to update the risk register with Phase 2 risks
- `gate-checklist` — use when preparing the Gate B review package

## Entry Criteria
- Gate A passed; project formally funded; team assembled

## Exit Criteria
- Inception Closure Pack signed off; Gate B passed
