# Phase Guide

## Phase 1 — Discovery & Framing

**Objective:** Validate the opportunity and form testable hypotheses.

**Duration:** 1-3 weeks

**Key Activities:**
- Frame the opportunity and articulate the problem
- Screen technical and commercial feasibility
- Validate the problem with evidence
- Map solution and AI hypotheses

**Agents:** opportunity-framing (1.1) → feasibility-screening (1.2) → problem-validation (1.3) → hypothesis-mapping (1.4)

**Key Artefacts:** Opportunity Brief, Feasibility Note, Hypothesis Canvas, Stakeholder Map

**Gate:** A — Opportunity Approval

---

## Phase 2 — Architecture & Planning

**Objective:** Design the solution and plan iterations.

**Duration:** 1-2 weeks

**Key Activities:**
- Design solution architecture and produce ADRs
- Plan iterations with sprint structure and milestones
- Build comprehensive risk register

**Agents:** solution-architecture (2.1) → iteration-planning (2.2) → risk-register (2.3)

**Key Artefacts:** Solution Brief, Architecture Decisions, Iteration Plan, Risk Register, Assumption Register

**Gate:** B — Architecture Approval

---

## Phase 3 — Iteration Design

**Objective:** Plan each sprint for execution.

**Duration:** 1-2 days (per sprint cycle)

**Key Activities:**
- Design sprint: goal, committed backlog, capacity
- Define acceptance criteria (BDD-style)
- Define test strategy

**Agents:** sprint-design (3.1) → acceptance-criteria (3.2) → test-strategy (3.3)

**Key Artefacts:** Sprint Backlog, Acceptance Criteria Catalog, Test Plan, DoD Checklist

**Gate:** C — Sprint Readiness (per sprint)

---

## Phase 4 — Build & Integrate

**Objective:** Implement, integrate, and quality-assure.

**Duration:** 1-4 weeks (per sprint)

**Key Activities:**
- Build features with code review and unit tests
- Integrate components with integration testing
- Implement AI/ML components and run experiments
- QA: regression, performance, defect management

**Agents:** feature-builder (4.1) → integration-engineer (4.2) → ai-implementation (4.3) → quality-assurance (4.4)

**Key Artefacts:** Feature Specs, Integration Test Record, AI Experiment Log, Model Card, Defect Log

**Gate:** D — Build Quality

---

## Phase 5 — Validate & Gate

**Objective:** End-to-end validation and release preparation.

**Duration:** 1-2 weeks

**Key Activities:**
- End-to-end functional testing and UAT
- AI model validation (performance, bias, safety, red-team)
- Assemble Gate E pack with all evidence
- Collect stakeholder sign-offs

**Agents:** functional-validation (5.1) → ai-model-validation (5.2) → gate-preparation (5.3) → stakeholder-review (5.4)

**Key Artefacts:** Functional Test Report, AI Validation Report, UAT Report, Gate E Pack

**Gate:** E — Release Readiness

---

## Phase 6 — Release

**Objective:** Deploy to production and stabilise.

**Duration:** 2-6 weeks (including hypercare)

**Key Activities:**
- Plan release: rollout strategy, go-live criteria, rollback triggers
- Execute deployment with real-time monitoring
- Lead hypercare: intensive monitoring, incident response

**Agents:** release-manager (6.1) → deployment-engineer (6.2) → hypercare-lead (6.3)

**Key Artefacts:** Release Plan, Deployment Record, Rollback Plan, Hypercare Report, Operations Handover

**Gate:** F — Operations & Hypercare Completion

---

## Phase 7 — Operate & Improve

**Objective:** Continuous operations, monitoring, and improvement.

**Duration:** Ongoing

**Key Activities:**
- Monitor SLAs and manage incidents
- Monitor AI model drift and retraining
- Drive continuous improvement cycles
- Close lifecycle iteration or retire

**Agents:** operations-monitor (7.1) → ai-ops-analyst (7.2) → continuous-improvement (7.3) → lifecycle-close (7.4) → retirement-planner (7.5)

**Key Artefacts:** Service Reports, AI Monitoring Reports, Retrospective Records, Improvement Backlog, Lifecycle Closure
